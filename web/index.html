Pipeline

Five steps. Data stays home. Only proof moves.

Every stage is engineered so that what crosses the trust membrane is opaque or aggregate — never a raw record.

01

Local signals

Each party derives anomaly signals inside its own perimeter: payout velocity, off-platform steering, link bursts. Computed under that org's own row-level security.

on-prem
crosses: nothing
02

OPRF commitments

Candidate signatures become opaque commitments under a jointly-governed key. No party can enumerate another's inputs.

oprf-ristretto255
crosses: commitments
03

Protected match

ECDH / DDH private-set-intersection runs across party agents. Coordinator relays blinded group elements and learns only the intersection. Non-matching tokens stay hidden.

ECDH_PSI_3PC
crosses: group elements
04

Federated risk score

Logistic regression over five aggregate features — trained offline, reproducible from seed. Not an LLM, not a random number.

phylax-risk 1.0.0
crosses: 5 aggregate features
05

Human-approved action

Smallest proportionate response — hold payout, quarantine listing, warn recipients. Human at target org approves. Nothing auto-executes.

Ed25519 receipt
crosses: decision hash
Audit

What is running — and what is honestly still a demo.

Security products earn trust by being precise about their own limits.

✓ Protocol is real

DDH-PSI executes locally; SecretFlow SPU ECDH_PSI_3PC in production. Non-matching tokens hidden by DDH assumption. Coordinator never sees a raw identifier.

✓ Model is trained

Logistic regression fitted offline by gradient descent, verified to reproduce from seed. Shipped with feature spec and params hash. Not an LLM.

✓ Zero raw records

raw_records_shared = 0 is a hard database invariant enforced by trigger. Never a message, identity, account number, watchlist, or vector enters the neutral plane.

! Topology is staged

Single-account demo with synthetic data. Proves protocol and product — not adversarial infrastructure isolation. Production: per-org projects with independent key material.
