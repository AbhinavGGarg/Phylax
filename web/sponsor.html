Native, not bolted on.

Phylax could not exist as a stack of loosely-wired services. The whole product is a trust boundary with a state machine, an audit ledger, and a live stream running through it — which is exactly the shape InsForge has.

The thesis

Phylax is built natively on InsForge. InsForge creates the trust boundaries between organizations, hosts the long-running protected-compute services, enforces access controls, persists the auditable decision ledger, streams the live incident response, stores private artifacts, and deploys the control plane. Phylax's own privacy-preserving computation runs on that foundation; InsForge makes that computation a usable, trusted product.

InsForge product → Phylax usage
InsForgelayerHow Phylax uses it
Auth + RLSboundary Creates the trust boundaries. 6 roles across 4 orgs; a partner can never read another partner's private signal batches or commitments. Every write is authorized from membership, not a client-supplied id.
Postgresledger Persists the auditable decision ledger. 13 tables behind a monotonic, server-enforced state machine; the audit + finding tables are append-only and raw_records_shared = 0 is a hard invariant.
Edge Functionsworkflow Enforces access controls & drives the workflow. 8 functions run the pipeline — submit → match → score → approve → receipt — with HMAC-SHA256 + timestamp window + single-use nonce replay protection on worker callbacks.
Custom Computecompute Hosts the long-running protected-compute services. Dockerized party compute services (Linux · Python 3.10) run each org's PSI + secure-aggregation share.
Realtimestream Streams the live incident response. run: and org: channels carry safe metadata only — phase, score, cardinality, the zero counter — with subscription gated by RLS.
Storageartifacts Stores private artifacts. A private phylax-artifacts bucket holds input packages, model artifacts, and signed receipts; the DB keeps only pointers + checksums.
Backend Branchingsafety Guards the boundary changes. A schema-only security branch validates RLS and state-machine edits before they reach the production control plane.
Sitesdeploy Deploys the control plane. The operator console and this site ship on InsForge Sites, served from the same project that governs the data.
