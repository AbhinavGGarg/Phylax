Phylax runs on InsForge.
Every single layer.

This is not InsForge for a login and a table. Phylax is a trust boundary between rival companies, a monotonic state machine, an append-only ledger, long-running cryptographic compute, and a live incident stream — and InsForge is every one of those things, in one project, behind one set of credentials. We didn't glue services together. We built on the platform that is already shaped like the product.

The thesis

Phylax is built natively on InsForge. InsForge creates the trust boundaries between organizations, hosts the long-running protected-compute services, enforces access controls, persists the auditable decision ledger, streams the live incident response, stores private artifacts, and deploys the control plane — the very page you're reading included.

Why it had to be InsForge

The platform is agent-native. So is the product.

It gives you the hard parts

Isolated Postgres with real triggers, edge functions, realtime, storage, long-running compute, backend branching, and deploys — behind one set of credentials. The parts that usually take a team and a month were already there.

The trust boundary is first-class

Phylax's entire premise is "prove the overlap, share nothing." InsForge Auth + RLS make that a property of the database, not a promise in a README. That's the difference between a demo and something a bank would trust.

It made a hard product shippable

Privacy-preserving multi-party computation, a human-approval workflow, and a tamper-evident ledger — built, deployed, and clickable by a judge. On InsForge that's a weekend. Anywhere else it's a quarter.

It's live, not localhost

Everything here runs on a real InsForge project right now. Open the console, hit Run Protected Sweep, and watch the state machine, the realtime stream, and the signed receipt do their jobs — no "trust me," no mock.
